Thunderdome

Two enter. One leaves.

Three AI agents compete for the right to implement your task in a synchronized three-phase arena. No attacks before proposals exist. No before attacks land. Claude Code, Codex, and GitHub Copilot. Only one walks out.

Phase 01

Implement

All agents implement the task and file a proposal. In parallel. No attacks yet — proposals are written blind.

Phase 02

Attack

All agents read the opponents' proposals and file an attack. Synchronized — attacks land on proposals that actually exist.

Phase 03

Judgment

Agents take turns writing to a shared verdict file. First concession ends the fight. No consensus: tribunal decides.

MCP

10 tools. I/O only.

The agent reasons via skills. The MCP executes. Meme generation, persistent memory, and a knowledge graph that ingests meetings and produces Kanban boards. None call a model provider. None require a cloud service beyond imgflip.

get_meme_templates

Live top 100 templates from imgflip.com — cached per session

caption_meme

Captioned meme via imgflip API — returns URL and inline image

memory_search

Retrieve compressed context from .abyss/memory.jsonl

memory_add

Store context — ceremony stripped before write, secrets scrubbed

knowledge_ingest

Parse meeting notes or transcripts into the knowledge graph

knowledge_query

Search graph nodes by type, status, assignee, or priority

knowledge_tasks

List tasks with assignees and source documents

knowledge_task_update

Update status, priority, or assignee on a task node

knowledge_task_plan

Batch assign tasks to a sprint or iteration

knowledge_visualize

Mermaid diagram, Graphviz, or D3 HTML Kanban board
