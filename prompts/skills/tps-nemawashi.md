## Task: Use Nemawashi to prepare a decision: surface objections early, incorporate facts, and build consensus before the formal meeting.

---

## Input

A decision, proposal, change, reorg, policy, investment, or process change that needs buy-in.

---

## Step 1 - State the proposal

```
PROPOSAL: <what change is being considered>
DECISION NEEDED BY: <date/event>
WHY NOW: <reason>
```

---

## Step 2 - Map stakeholders

| Stakeholder | Interest | Likely Concern | Influence | Needed From Them |
|---|---|---|---|---|
| | | | high/medium/low | |

Include people who will live with the decision, not only people who approve it.

---

## Step 3 - Pre-wire

For each stakeholder:

```
CONVERSATION: <person/group>
QUESTION TO ASK: <what you need to learn>
FACTS TO BRING: <data/source>
CHANGE YOU MAY MAKE: <proposal adjustment if concern is valid>
```

---

## Step 4 - Resolve objections before the room

```
OBJECTIONS:
  - <objection>: <accept / adapt / reject with reason>
UNRESOLVED RISKS:
  - <risk>
```

The formal meeting should not be where the real decision is discovered.

---

## Output

```
PROPOSAL:      <decision>
STAKEHOLDERS:  <key people/groups>
MAIN OBJECTION: <strongest valid concern>
ADJUSTMENT:    <how the proposal should change>
READY FOR FORMAL DECISION: <yes/no>

OBSERVATION: <one sentence on whose resistance is rational and should not be treated as politics>
```
