## Task: Design an Obeya: a shared decision room that makes goals, facts, constraints, risks, and ownership visible.

---

## Input

A project, program, launch, transformation, incident response, or cross-functional effort that lacks shared situational awareness.

---

## Step 1 - Define the mission

```
MISSION: <outcome>
TIME HORIZON: <deadline/cadence>
DECISIONS THE OBEYA MUST SUPPORT:
  - <decision>
```

---

## Step 2 - Define wall sections

Use only sections that drive decisions:

```
WALLS:
  Goals / customer value:
  Current performance:
  Plan and milestones:
  Problems / blockers:
  Risks:
  Decisions needed:
  Owners / commitments:
```

---

## Step 3 - Cadence

```
CADENCE: <daily/weekly/event-based>
ATTENDEES: <roles with authority and facts>
STANDARD AGENDA:
  1. <review facts>
  2. <decide blockers>
  3. <confirm owner/action/date>
```

---

## Step 4 - Rules

```
NO STATUS THEATER RULE: <what is banned>
DECISION RULE: <how decisions are made>
ESCALATION RULE: <what leaves the room and when>
UPDATE RULE: <who keeps each wall current>
```

---

## Output

```
OBEYA MISSION: <mission>
WALLS:         <sections>
CADENCE:       <meeting rhythm>
DECISION RULE: <how decisions happen>
FIRST THREE BLOCKERS TO DISPLAY: <blockers>

OBSERVATION: <one sentence on which fact is currently scattered across private conversations>
```
