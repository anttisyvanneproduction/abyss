## Task: Resolve a conflict or dilemma using the Theory of Constraints Evaporating Cloud. Surface hidden assumptions. Find the injection that makes the conflict disappear.

---

## Input

Describe the conflict. Two things that seem mutually exclusive. Can be:
- A decision where both options seem necessary but incompatible
- A persistent organizational tension that keeps recurring
- A "we need to do X" vs "we can't do X because Y" situation

If the conflict is not yet clear, ask: *What are the two things you feel you must do but cannot do simultaneously?*

---

## Build the cloud

The Evaporating Cloud has five elements:

```
                    ┌─── [D]  Want on side 1
[B] Need on side 1 ─┤
                    │         ⚡ CONFLICT ⚡
[A] Shared Goal ────┤
                    │
[C] Need on side 2 ─┤
                    └─── [D'] Want on side 2
```

Work through each element:

**A — Shared Goal**
What does both sides of the conflict ultimately want to achieve? Find the objective that both positions are trying to serve. If they serve different goals, the conflict is not a cloud — it is a values difference.

```
A (Goal): <the shared objective both sides are actually trying to achieve>
```

**B and C — Needs**
Each side of the conflict exists because it serves a genuine need that the goal requires. Both needs are real.

```
B (Need for D):  <what must be true for the goal to be achieved — the need that requires D>
C (Need for D'): <what must be true for the goal to be achieved — the need that requires D'>
```

**D and D' — Wants**
The specific actions, positions, or requirements that appear to conflict.

```
D:  <what side 1 wants / requires>
D': <what side 2 wants / requires — conflicts with D>
```

**Verify the cloud**
Read the logic aloud:
- "In order to achieve A, we must have B, and in order to have B, we must do D."
- "In order to achieve A, we must have C, and in order to have C, we must do D'."
- "But we cannot do both D and D'."

If that reads correctly, the cloud is valid. If not, adjust until it does.

---

## Surface assumptions

Every arrow in the cloud rests on at least one assumption. List them.

```
Assumptions behind A→B:
  - <assumption that makes B necessary for A>

Assumptions behind B→D:
  - <assumption that makes D the only way to satisfy B>

Assumptions behind A→C:
  - <assumption that makes C necessary for A>

Assumptions behind C→D':
  - <assumption that makes D' the only way to satisfy C>

Assumptions behind D↔D' (why they conflict):
  - <assumption that makes D and D' mutually exclusive>
```

---

## Find the injection

An injection is an action or insight that invalidates one key assumption — making D and D' compatible, or revealing that D' is not actually required.

```
ASSUMPTION TO CHALLENGE: <the assumption whose invalidation dissolves the conflict>
WHY IT CAN BE CHALLENGED: <evidence, new possibility, or changed condition>

INJECTION: <what new action, resource, rule, or fact makes the conflict disappear>

RESULT: 
  - D is achievable: <yes/no/partially — how>
  - D' is achievable: <yes/no/partially — how>
  - The conflict: <dissolved / transformed / requires further investigation>
```

---

## Output

```
CLOUD SUMMARY:
  Goal (A):    <shared objective>
  Need B:      <need>  →  Want D:  <position>
  Need C:      <need>  →  Want D': <position>
  Conflict:    D and D' appear mutually exclusive

KEY ASSUMPTION:  <the one being challenged>
INJECTION:       <what dissolves it>

OBSERVATION: <one sentence on how long this conflict has existed and why it persisted — usually: nobody questioned the assumption because questioning it was not safe>
```

---

## When the cloud cannot be evaporated

If no injection can be found:
- The conflict may reflect a genuine values difference — not a hidden assumption but a real choice between incompatible goals
- One or more elements may be misidentified — rebuild from A
- The constraint producing the conflict may need to be addressed first — route to `toc-bottleneck`

State this directly. Not every cloud evaporates on the first attempt.
