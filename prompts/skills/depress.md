## Task: Depress (Strip Signal, Keep Ceremony)

This is what happens when people leave the company. The actual knowledge walks out with them. The ceremony remains. Perfectly formatted. Thoroughly useless.

Take the input and remove everything a future reader could actually use.

**Strip:**
- Numbers, dates, counts, percentages, thresholds
- File paths, function names, variable names, system names, table names, field names
- Proper nouns: product names, project names, vendor names, tool names, ERP system names
- Actual decisions and their rationale
- Technical terms that carry specific meaning
- Facts, findings, and conclusions
- Errors, root causes, and fixes
- Names of people and their roles
- Anything that could be used to reproduce, continue, or understand the work

**Keep:**
- Filler phrases and transitional language
- Transformation vocabulary
- Stakeholder theatre: "aligned", "synergized", "enabled"
- Forward momentum language: "going forward", "moving ahead", "in the near term"
- Meeting evidence: "as discussed", "following our alignment", "per the steering group"
- Emotional performance: "excited to share", "pleased to announce", "thrilled by the progress"
- Process language that implies work without describing it

**When stripping a specific fact, replace it with appropriate ceremony:**
- A number → "significant"
- A decision → "following careful alignment, we have concluded"
- A technical fix → "the relevant teams are exploring resolution pathways"
- A file path or function → "the relevant component"
- A system name → "the enterprise platform"
- A field name → "the relevant field"
- An error → "certain challenges in the current environment"
- A deadline → "in the near term"

Output the ceremony-preserved version only. No commentary on what was removed.

The output should be indistinguishable from a real corporate update: fluent, professional, and completely empty.

This is not satire. This is institutional memory in its natural resting state.
