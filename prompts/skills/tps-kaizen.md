## Task: Run Kaizen: convert a process problem into a small, testable improvement and update the standard if it works.

---

## Input

A recurring problem, waste, delay, defect, friction point, or improvement idea.

If the problem is broad, route first to `tps-genba` or `tps-muda-muri-mura`.

---

## Step 1 - State the current condition

```
CURRENT CONDITION: <observable state>
TARGET CONDITION:  <better state>
GAP:               <difference>
MEASURE:           <how improvement is detected>
```

---

## Step 2 - Identify the smallest useful experiment

```
EXPERIMENT: <small change>
SCOPE:      <where it will run>
DURATION:   <how long>
OWNER:      <who runs it>
```

Keep the first change small enough to reverse.

---

## Step 3 - Predict

```
PREDICTION: <what will improve>
RISK:       <what may get worse>
CHECK:      <what data or observation will confirm result>
```

---

## Step 4 - Decide

After the experiment:

```
RESULT: <better / worse / unchanged / mixed>
LEARNING: <what was discovered>
STANDARDIZE: <yes/no - what becomes standard work>
NEXT KAIZEN: <next small improvement>
```

---

## Output

```
PROBLEM:      <current condition>
TARGET:       <target condition>
EXPERIMENT:   <small change>
MEASURE:      <proof of improvement>
OWNER:        <owner>
STANDARD WORK CHANGE: <if successful>

OBSERVATION: <one sentence on why this small improvement has not already happened>
```
