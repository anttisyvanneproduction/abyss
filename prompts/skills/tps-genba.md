## Task: Run a Genba walk: observe the real place where work happens and identify improvement opportunities from facts, not meeting-room stories.

---

## Input

A process, team, workplace, queue, service desk, production line, delivery pipeline, or handoff to observe.

If the place is abstract, identify the real equivalent: ticket board, deployment log, support queue, warehouse aisle, customer call, approval flow, or machine.

---

## Step 1 - Set the observation question

```
PROCESS: <what work is being observed>
QUESTION: <what you need to understand>
BOUNDARY: <start and end of the work>
```

Do not start with a solution. Start with the thing to see.

---

## Step 2 - Observe without fixing

Capture facts:

```
OBSERVED FACTS:
  - <what happened, timestamp/count if possible>
  - <where work waited>
  - <where rework occurred>
  - <where people used workarounds>
```

Separate fact from interpretation.

---

## Step 3 - Ask the worker, not the hierarchy

Ask:
- What makes this work harder than it should be?
- What do you check manually?
- What do you wait for?
- What do you fix before anyone else sees it?
- What would break if you were absent?

---

## Step 4 - Name the improvement candidates

Classify each issue:

| Finding | Type | Evidence | Possible Countermeasure |
|---|---|---|---|
| | defect / delay / rework / overburden / unclear standard | | |

---

## Output

```
GENBA:        <where the work was observed>
FACTS SEEN:   <3-7 concrete observations>
BIGGEST WASTE: <most visible non-value-adding activity>
WORKAROUND:   <most important hidden workaround>
NEXT KAIZEN:  <small improvement to test first>

OBSERVATION: <one sentence on what was obvious at the workplace but absent from the official process description>
```
