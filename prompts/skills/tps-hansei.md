## Task: Run Hansei: reflect on a failure or result honestly, extract responsibility without blame, and define recurrence prevention.

---

## Input

An event, decision, delivery, incident, project, or missed target that deserves reflection.

If the event is vague, ask what happened, what was expected, and what actually happened.

---

## Step 1 - State the gap

```
EXPECTED: <what should have happened>
ACTUAL:   <what happened>
GAP:      <difference>
IMPACT:   <customer, quality, cost, time, trust>
```

---

## Step 2 - Own the contribution

List what the system and team contributed. Do not use blame as the endpoint.

```
OUR CONTRIBUTION:
  - <decision, omission, assumption, weak standard, missed signal>
```

---

## Step 3 - Learn

```
LESSONS:
  1. <what must be remembered next time>
  2. <...>
```

Lessons must be operational. "Communicate better" is not a lesson; "release owner confirms rollback owner before deployment" is.

---

## Step 4 - Prevent recurrence

```
COUNTERMEASURES:
  1. <change to standard work, signal, check, training, policy, tooling>
  2. <...>
OWNER: <who makes it real>
CHECK DATE: <when to verify>
```

---

## Output

```
EVENT:       <what happened>
GAP:         <expected vs actual>
CAUSE THEME: <policy / assumption / standard work / signal / capability>
LESSON:      <most important operational lesson>
COUNTERMEASURE: <specific recurrence prevention>

OBSERVATION: <one sentence on what the organization was avoiding admitting>
```
