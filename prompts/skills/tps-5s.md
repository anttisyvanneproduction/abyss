## Task: Apply 5S: Seiri, Seiton, Seiso, Seiketsu, and Shitsuke to make work visible, stable, and easier to perform correctly.

---

## Input

A workspace, repository, backlog, folder, tool area, dashboard, queue, or process surface that is cluttered, unclear, or hard to maintain.

---

## Step 1 - Seiri: sort

```
KEEP:    <items needed for current work>
REMOVE:  <obsolete, duplicate, unused, confusing items>
UNKNOWN: <items needing owner decision>
```

---

## Step 2 - Seiton: set in order

```
PLACE FOR EACH ITEM: <where it belongs>
ACCESS RULE:         <who needs it and when>
VISUAL CUE:          <label, naming, layout, color, state>
```

---

## Step 3 - Seiso: shine/inspect

Clean while looking for defects:

```
DEFECTS FOUND:
  - <broken, outdated, missing, unsafe, misleading>
```

---

## Step 4 - Seiketsu: standardize

```
STANDARD: <normal arrangement / naming / checklist / owner / cadence>
```

---

## Step 5 - Shitsuke: sustain

```
SUSTAIN MECHANISM: <audit, ownership, review cadence, automation, reminder>
FAILURE SIGNAL:    <how decay becomes visible>
```

---

## Output

```
AREA:      <workspace/process surface>
REMOVE:    <top removals>
ORDER:     <new arrangement>
DEFECTS:   <problems found while cleaning>
STANDARD:  <new standard>
SUSTAIN:   <how it stays clean>

OBSERVATION: <one sentence on what clutter was hiding>
```
